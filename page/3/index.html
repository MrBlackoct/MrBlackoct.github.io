Javascript中Array(数组)对象的方法

方法

  1. concat() 方法用于连接两个或多个数组
    • 接收参数arrayX.可以是具体值或数组对象.可以是任意多个.
    • 该方法不会改变原数组,而是返回一个被连接数组的副本.
  2. join() 方法把数组中所有元素放入一个字符串
    • 接收参数separator(可选).指定要使用的分隔符,若省略,则使用 , .
    • 该方法不会改变原数组.
  3. pop() 方法删除并返回数组的最后一个元素
    • 返回它删除的元素的值
    • 如果数组已经为空,则不会改变数组,返回undefined.
    • 该方法会改变原数组.
  4. push() 方法向数组末尾添加一个或多个元素,并返回新的长度
    • 接收参数newelement,数量为1个或多个.
    • 返回值为新的数组长度.
    • 该方法会改变原数组.
  5. reverse() 方法用于点到数组中元素的顺序
    • 该方法会改变原数组,不创建新的数组.
  6. shift() 方法删除并返回数组第一个元素
    • 返回值为第一个元素的值.
    • 若数组为空则返回undefined.
    • 该方法会改变原数组.
  7. slice() 方法从已有数组中返回选定元素
    • 接收2个参数start(必需)/end(可选).start规定从何处开始选取;end规定在何处结束.
    • 如果start/end为负数,则从数组末尾开始计算.
    • start与end制定的都是每个元素槽位的后侧边缘,即:

      //此处有图
    • 该方法不会改变原数组,返回一个新的数组.
  8. sort() 方法用于对数组元素排序
    • 接收参数sortby,规定排序的顺序.此参数必须是函数.若参数为空,则按照字符编码顺序进行排序.
    • 该方法会改变原数组,返回对原数组的引用,不生成副本.
    • 比较数有两个参数: a/b. a/b是数组中随机的元素.

      简而言之,

      return a - b : 升序排列

      return b - a : 降序排列
  9. splice() 方法向/从数组中添加/删除项目,然后返回被删除的项目.
    • 接收三种参数index(必需)/howmany(必需)/item1,item2,…(可选).index规定要操作的位置,howmany规定要删除的数量,item为要在该位置添加的新项目.
    • index为负数时,从数列末尾计算位置.如果howmany为0则不删除,即只添加item.
    • 返回值为被删除项目组成的数组.
    • 该方法会改变原数组.
  10. tostring() 方法把数组转换成字符串并返回结果.
    • 返回值与没有参数的join()方法返回的字符串相同.
    • 该方法会改变原数组.
  11. unshift() 方法可向数组开头添加一个或多个元素,并返回新的长度.
    • 接收1个或多个参数.
    • 返回值为新的数组长度
    • 该方法会改变原数组,不创建副本.

对于原数组不同操作的方法的横向比较

  1. 改变原数组的方法:
    pop() / push() / unshift() / reverse() / shift() / splice() / sort()
  2. 不改变原数组的方法:
    concat() / join() / slice()

CSS浮动与清除

发表于 2017-12-07

CSS浮动与清除

浮动

  • 浮动的框可以向左或向右移动，直到它的外边缘碰到包含框或另一个浮动框的边框为止。
  • 由于浮动框不在文档的普通流中，所以文档的普通流中的块框表现得就像浮动框不存在一样。
  • 由于不在普通流中,浮动会导致父元素的高度坍塌
  • float属性:left/right/none/inherit

清除

  1. 添加额外标签

    缺点:添加无意义的标签,有违结构与表现的分离.
  2. 使用 br标签和其自身的 html属性

    缺点:添加无意义的标签,有违结构与表现的分离.
  3. 父元素设置 overflow：hidden

    缺点：内容增多时候容易造成不会自动换行导致内容被隐藏掉，无法显示需要溢出的元素.
  4. 父元素设置 overflow：auto 属性
  5. 父元素设置浮动
    缺点：使得与父元素相邻的元素的布局会受到影响，不可能一直浮动到body.
  6. 父元素设置display:table
    缺点：盒模型属性改变.

  7. 使用:after伪元素

    • U+200B 是一个Unicode字符里的“零宽度空格”,所以可以省略掉 visibility: hidden; ,替换为content:”200B”

BFC–Block Formatting Contexts （块级格式化上下文）

  1. 触发条件
    • float除none以外的值
    • overflow除visible以外的值(hidden/auto/scroll)
    • display = table/inline-block
    • position = absolute;fixed
  2. BFC的特性
    • BFC会阻止外边距增叠加
    • BFC不会重叠浮动元素
    • BFC通常可以包含浮动

px,em,rem & CSS选择器优先级

px/em/rem

  • px: 绝对大小.像素.
  • em: 相对长度单位,基于父级遗传字体大小计算.
  • rem: 相对长度单位,基于HTML根元素字体大小计算.
  • 任意浏览器的默认字体高都是16px.所有未经调整的浏览器都符合: 1em=16px.为了方便计算,可以将body中字体大小设置为font-size = 62.5%,则1em = 10px.rem同理.

AngularJS_生命周期 & 模块

NG生命周期

  1. 编译阶段:
    根据JS代码编译指令,返回一个模板函数.在返回模板函数之前,DOM树还没有进行数据绑定,此时操作DOM开销很小.
  2. compile/link(二者互斥,如果同时声明则返回compile的返回值)
    编译函数负责对模板DOM转化.链接函数将作用域和DOM连接.

NG模块

  1. 配置块 .config()
    • 以声明和注册的顺序执行.
    • 只有少数几种类型的对象可以被注入到config()中,如提供者和常量.
    • 只能用provider()构建服务.
  2. 运行块 .run()
    • 通常用来注册全局的事件监听器.

Javascript的函数提升 & 回调函数 & 箭头函数

JavaScript的函数提升

  1. 函数语句随函数主体一起被提升到执行上下文的顶部.可以在函数创建之前使用作为语句创建的函数.

  2. 函数表达式在创建之前不能使用,只有声明部分会被提升,赋值发生在创建函数行.

Javascript的回调函数(callback())

函数A作为参数(函数引用)传递到另一个函数B中,且函数B会执行函数A,则A是B的回调函数.

Javascript的箭头函数

  1. 来自于ES6(ES2015)

  2. 形式:

    • 不引入参数

    • 引入1个参数

    • 引入多个参数

    • 如果函数体内有更多语句执行则不能省略return

    • 如果要返回对象,应(),以示和函数{}区分

  3. 箭头函数的this总是指向词法作用域

  4. 没有原型;没有参数对象;不能作为构造函数

Javascript的词法作用域

  1. 含义:
    当定义了一个函数后,当前的作用于就会被保存下来,并且成为函数内部状态的一部分.
  2. 规则:
    • 函数允许访问函数外的数据
    • 如果当前作用于中存在某变量名,不考虑外部同名变量
    • 整个代码结构中只有函数可以限定作用域
    • 作用域中首先使用提升规则分析
  3. 作用域链
    本域中无定义则向上查找,顶级域(全局变量)中无定义则为is not defined.同级链中不可互相查找

Javascript this关键字

  • JS的this不同于其他语言,JAVA/C#之类的this一般是在编译时绑定,指向当前对象;而JS的this是在运行时绑定,这是JSthis关键字具备多重含义的本质原因.
  • this可以是全局对象/当前对象/任意对象,取决于函数的调用方式.

  • 判断流程:

    1. 函数是否由new进行调用
      1. 是 –> this指向新创建的对象
      2. 否
        1. 函数是否使用 . 调用
          1. 是 –> this指向 . 之前的对象
          2. 否 –> this指向全局对象

记一次走弯路的小经历

第一次参与了公司项目的维护修改工作.一开始显得笨手笨脚,花了很多很多很多时间在搭建环境上.这些都先按下不提,先留一下一个走弯路的小经历.

两次选择使用if判断的事件.

接收后端传入参数来区分该用户是否拥有查看部级/所级通告的权限,根据权限来设置按钮的显示/隐藏.相关参数为数组形式:

isDeptLeader = true代表该用户为部门领导,可以查看本部门公告;isAdmin = true代表该用户为院级领导(大领导),可以查看所有部门公告.两者都为false代表该用户为普通职员,可以查看本部门和本所级公告.

因为整体使用AngularJS,自然想到了使用ng-show/ng-hide来控制按钮的显示/隐藏.于是在.constructor中写了一个使用if的判断函数,当时的思路是

|false |ture |the Dept |
|true |false |All Detp |
|false |false |the Dept & the Station|
于是打算写一个使用一个if嵌套另一个if的代码块,定义三个变量给ng-show/hide作为参数来实现.

看后惊讶于自己的愚蠢与思维的原始和固化,由于自己代码量确实极度匮乏,经验真的少,遇到需要判断的情况就毫不犹豫不假思索地使用if语句来编写.

第二次情况其实很相似,接收后端传入参数来区分该用户是否拥有编辑通告的权限,并据此控制编辑相关按钮的显示/隐藏.相关参数也是作为数组传入:

FirstAuth = true代表该用户拥有所级通知公告的编辑权限;SeconAuth = true代表该用户拥有部级公告的编辑权限.

这次的情况稍微复杂一些.

第一,界面中可能存在内容面板的情况有三种:部级 + 部级 / 所级 / 部级 + 所级.

第二,该用户可能拥有的编辑权限也有三种情况:部级 / 所级 / 有所级没有部级 / 有部级没有所级 / 都没有.

刚开始拿到这个需求时,考虑到每次切换内容面板也要判断编辑相关按钮是否显示,内容面板会由于点击事件来回切换,再加上前几天在编写另一个自己的小应用中曾经用到了AngularJS的$watch()方法,所以打算使用$watch()来监控权限,并再次使用if语句来判断在每次点击加载内容面板时判断按钮是否显示.

造成这次我走弯路的原因除开经验不足外,还有两个原因.

第一,对整个项目代码和AnguJS的理解不够,$watch()应该用来监测会实时变化的数据,而其实作为通过调取服务接口从后端拿来的权限参数在用户登入,页面加载完成后就已经获取到且不会再变化了.

第二,没有简单透彻的审视这个需求,其实要实现这个需求要做的只是确定编辑相关按钮什么时候显示.简单来讲,要显示编辑相关按钮要具备的条件是:

  • 此时显示部级内容面板 && 拥有部级编辑权限
  • 此时显示所级内容面板 && 拥有所级编辑权限

仅此而已.

通过这两件事,我以后应该注意:

  • 对于布尔型事件,要特别留意,切入其本质进行解决.
  • 不要一遇到需要判断的事件就直接不假思索用if.
  • 对于项目代码要尽可能的熟悉.
