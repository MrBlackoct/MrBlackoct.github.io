三次握手 & 四次挥手

三次握手(由客户端执行connect来触发)

  1. Client将标志位SYN置为1,随机长生一个值seq = J,将该数据包发送给Server,Client进入SYN_SENT状态,等待Server确认.
  2. Server收到数据包后由SYN = 1知道Client请求建立连接,Server将标志位SYN和ACK都置为1,ack(number) = J + 1,随机产生一个值seq = k,将数据包发送给Client以确认连接请求,Server进入SYN_RECV状态.
  3. Client收到确认后,检查ack是否为J + 1,ACK是否为1.如果正确,则将标志位ACK置为1,ack = k + 1.将数据包发送给Server,Server检查ack是否位k + 1,如果正确则连接建立成功,Client和Server进入ESTABLISH状态,完成三次握手,随后Client和Server之间可以传输数据.

四次挥手(由客户端 / 服务端执行close触发)

由于TCP连接时全双工的,一次每个方向都必须单独进行关闭.这一原则是当一方完成数据发送任务后,发送一个FIN来终止这一方向上的连接,收到一个FIN只是意味着这一方向上没有数据流动了,即不会再收到数据了,但是这个TCP连接上仍然能够发送数据,直到这一方也发送了FIN.首先进行关闭的一方执行主动关闭,另一方执行被动关闭.

  1. Client发送一个FIN,用来关闭Client到Server的数据传输,Client进入FIN_WAIT_1状态.
  2. Server收到FIN后,发送一个ACK给Client,确认序号为收到序号+1.Server进入CLOSE_WAIT状态.
  3. Server发送一个FIN,用来关闭Server到Client的数据传输,Server进入LAST_ACK状态.
  4. Client收到FIN后,进入TIME_WAIT状态,接着发送一个ACK给Server,确认序号为收到序号+1,Server进入CLOSED状态.
  5. 四次挥手.jpg

为什么建立连接时三次,关闭是四次?

这是因为服务端在LISTEN状态下,收到建立连接的请求报文后,把ACK和SYN在一个报文里发给客户端.而关闭连接时,当收到对方的FIN报文时,仅仅标识对方不在发送数据了,但是还能接收数据,己方数据也未必全部发给对方了,所以己方可以立即close,也可以发送一些数据之后,在发送FIN报文给对方,标识统一现在关闭连接.因此,己方ACK和FIN一般都会分开发送.
