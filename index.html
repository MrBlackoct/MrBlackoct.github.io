发表于 2017-12-19

HTTP相关汇总

HTTP请求报文与响应报文格式

  • 请求报文包含3部分:
    1. 请求行:包含HTTP版本 / 状态码 / 状态码的原因短语
    2. 请求首部字段
    3. 请求内容实体
  • 响应报文包含3部分:
    1. 状态行:包含HTTP版本 / 状态码 / 状态码的原因短语
    2. 响应首部字段
    3. 响应实体内容

常见HTTP首部字段

  1. 通用首部字段
    • Date : 报文创建时间
    • Connection : 连接的管理
    • Cache-Control : 缓存的控制
    • Transfer-Encoding : 报文主体的编码方式
  2. 请求首部字段
    • Host : 资源所在服务器
    • Accept : 可处理的媒体类型
    • Accept-Charset : 可接收的字符集
    • Accept-Encoding : 可接收的内容编码
    • Accept-Language : 可接收的自然语言
  3. 响应首部字段
    • Accept-Ranges : 可接收的字节范围
    • Location : 令客户端重新定向到的URI
    • Server : HTTP服务器的安装信息
  4. 实体首部字段
    • Allow : 资源可支持的方法
    • Content-Type : 实体主类的类型
    • Content-Language : 实体主体的自然语言
    • Content-Length : 实体主体的字节数
    • Content-Range : 实体主体的位置范围,一般用于发出部分请求时使用

常见的HTTP方法

  1. GET : 用于请求访问已经被URI(统一资源标识符)识别的资源,可以通过URL传参给服务器.
  2. POST : 用于传输信息给服务器,主要功能与GET类似,但一般推荐使用POST.
  3. PUT : 传输文件.报文主体中包含文件内容,保存到对应的URL位置.
  4. HEAD : 获得报文首部,与GET类似,但不返回报文主体,一般用于验证URL是否有效.
  5. DELETE : 删除文件.与PUT相反,删除对应URL位置的文件.
  6. OPTION : 查询相应的URI支持的方法.

GET / POST

  1. GET重点在从服务器上获取资源,POST重点在向雾浮起发送数据.
  2. GET传输数据是通过URL请求,以field(字段) = value的形式置于URL后,并用 ? 链接,多个数据请求间用 & 连接.这个过程用户可见.
  3. GET受URL长度限制,传输数据量小但效率高.POST可以传输大量数据,所以上传文件只能用POST方式.
  4. GET不安全,因为URL可.POST安全性较高.
  5. GET只支持ASCII字符.向服务器中传中文字符可能会乱码.POST支持标准字符集,可以正确传输中文.

常见HTTP状态码

1 —- 信息,服务器收到请求,需要请求者继续执行操作.
2 —- 成功,操作被成功接收并处理.
3 —- 重定向,需要进一步的操作以完成请求.
4 —- 客户端错误,请求包含语法错误或无法完成请求.
5** —- 服务器错误,服务器在处理请求的过程中发生错误.

200 : 请求被正常处理.
204 : 请求被受理但没有资源可以返回.
206 : 客户端只是请求资源的一部分,服务器只对请求的部分资源执行GET方法,响应报文中通过Content-Range指定范围的资源.
301 : 永久性重定向.
302 : 临时重定向.
303 : 与302相似,但它希望客户端在请求一个URL的时候能通过GET方法重定向到另一个URL上.
304 : 发送附带条件的请求时,条件不满足返回,与重定向无关.
307 : 与302相似,但强制要求使用POST方法.
400 : 请求报文语法有误,服务器无法识别.
401 : 请求需认证.
403 : 请求的对应资源禁止被访问.
404 : 服务器无法找到对应资源.
500 : 服务器内部错误.
503 : 服务器正忙.

HTTP的缺点与HTTPS

  • HTTP的缺点
    1. 通信使用明文不加密,内容可能被窃听.
    2. 不验证通信方身份,可能遭到伪装.
    3. 无法验证报文完整性,可能被篡改.
  • HTTPS就是HTTP加上加密处理(一般是SSL安全通信线路) + 认证 + 完整性保护.

输入一个网址并跳转后发生了什么

  1. 域名解析.检查顺序为:
    浏览器自身的DNS缓存 -> OS自身DNS缓存 -> 读取host文件 -> 本地域服务器 -> 权限域名服务器 -> 根域名服务器
  2. TCP3次握手建立连接
  3. 建立连接发起HTTP请求
  4. 服务器端响应HTTP请求,浏览器得到HTTP请求的内容.
  5. 浏览器解析HTML代码,并请求代码中的资源.
  6. 浏览器对页面进行渲染.

MVC & 设计模式 & 面向对象三大特征

MVC原理和MVC模式的优缺点

  1. MVC原理
    • mvc是一种程序开发设计模式,它实现了显示模块与功能模块的分离.提高了程序的可维护性、可移植性、可扩展性与可重用性，降低了程序的开发难度.它主要分模型、视图、控制器三层.
    • Model(模型)是应用程序中用于处理应用程序数据逻辑的部分.它是应用程序的主体部分，主要包括业务逻辑模块（web项目中的Action,dao类）和数据模块（pojo类）.模型与数据格式无关，这样一个模型能为多个视图提供数据.由于应用于模型的代码只需写一次就可以被多个视图重用，所以减少了代码的重复性.
    • View(视图)是用用程序中处理数据显示的部分,通常视图是依赖模型数据创建的.它是用户与之交互的界面、在web中视图一般由jsp,html组成.
    • Controller(控制器)是应用程序中处理用户交互的部分.通常控制器负责从视图读取数据,控制用户输入并向模型发送数据,这个过程中控制器不做任何处理只是起到了一个连接的做用.
  2. MVC的优点
    • 可以为一个模型在运行时同时建立和使用多个视图.变化-传播机制可以确保所有相关的视图及时得到模型数据变化，从而使所有关联的视图和控制器做到行为同步.
    • 视图与控制器的可插接性允许更换视图和控制器对象,而且可以根据需求动态地打开或关闭,甚至在运行期间进行对象替换.
    • 模型的可移植性.因为模型是独立于视图的,所以可以把一个模型独立地一直到新的平台.需要做的只是在新平台上对视图和控制器进行修改.
    • 潜在的框架结构.可以基于此模型建立应用程序框架,不仅仅是在设计界面的设计中.
  3. MVC的不足
    • 增加了系统和实现的复杂性.对于简单的界面,严格遵循MVC,会使模型/视图与控制器分离,会增加结构的复杂性,并可能产生过多的更新操作,降低运行效率.
    • 视图与控制器之间的联系过于紧密.视图与控制器是相互分离但确实紧密联系的部件,视图没有控制器的存在,其应用是很有限的,反之亦然,这就妨碍了它们的独立重用.
    • 视图对模型数据的访问效率低.依据模型操作接口的不同,视图可能需要多次调用才能获得足够的显示数据.对未变化数据的不必要的频繁访问也将损害性能.
    • 目前,一般高级的界面工具构造器不支持模式.

设计模式

  1. 基本原则及六大原则
    • 基本原则–开闭原则
      对扩展开放,对修改封闭.在程序需要进行拓展的时候,不能去修改原有的代码,而是扩展原有代码,实现热拔插的效果.
    • 单一职责原则
      不要存在多于一个导致类变更的原因.一个类只负责一项职责,否则就应该把类拆分.
    • 里氏替换原则
      任何基类可以出现的地方,子类一定可以出现.里氏替换原则中,子类对父类方法尽量不要重写和重载.
    • 依赖倒转原则
      面向接口编程,依赖于抽象而不依赖于具体.写代码要用到具体类时,不与具体类交互,而与具体类的上层接口交互.
    • 接口隔离原则
      客户端不应该依赖它不需要的接口;一个类对另一个类的依赖应该建立在最小的接口上.
    • 迪米特法则(知道最少原则)
      一个对象应该对其他对象保持最少的了解.(尽量降低类与类之间的耦合).
    • 合成复用原则
      尽量首先使用合成/聚合的方式,而不是使用继承.
  2. 模式举例
    • 创建型模式
      工厂方法模式 / 抽象工厂模式 / 单例模式 / 建造者模式 / 原型模式
    • 结构性模式
      适配器模式 / 代理模式 / 装饰者模式 / 外观模式 / 桥接模式 / 组合模式 / 享元模式
    • 行为型模式
      策略模式 / 模板方法模式 / 观察者模式 / 迭代子模式 / 责任链模式 / 访问者模式
    • 其他
      并发模式 / 线程池模式

  • 继承
  • 多态

=== 与 == 的区别

=== 叫做严格相等运算符, == 叫做相等运算符

  1. 严格相等运算符规则
    • 不同类型的值:
      直接返回false.
    • 同一类型的原始值:
      值相同返回true,不同返回false.
    • 同一类型的复合值:
      两个复合类型(对象/数组/函数)比较时比较它们是否指向同一对象.
    • undefined/null:
      各自与自身严格相等.
  2. 相等运算符运算规则
    在比较相同类型数据时与严格相等运算符完全相同,在比较不同类型数据时先将数据类型转化再比较,转化规则如下:
    • 原始类型的值会转化为数值类型再比较.字符串和布尔值(0/1/对象本身/NaN)
    • 对象与原始类型值比较:
      对象先转化为原始类型值再比较.
    • undefined/null:
      与其他类型值相比是都为false,两者互相比较时返回true.

JS数据类型:

  • 基础类型:
    字符串/数字/布尔/NULL/UNDEFINED
  • 复合类型:
    数组/对象

Javascript中Array(数组)对象的方法

方法

  1. concat() 方法用于连接两个或多个数组
    • 接收参数arrayX.可以是具体值或数组对象.可以是任意多个.
    • 该方法不会改变原数组,而是返回一个被连接数组的副本.
  2. join() 方法把数组中所有元素放入一个字符串
    • 接收参数separator(可选).指定要使用的分隔符,若省略,则使用 , .
    • 该方法不会改变原数组.
  3. pop() 方法删除并返回数组的最后一个元素
    • 返回它删除的元素的值
    • 如果数组已经为空,则不会改变数组,返回undefined.
    • 该方法会改变原数组.
  4. push() 方法向数组末尾添加一个或多个元素,并返回新的长度
    • 接收参数newelement,数量为1个或多个.
    • 返回值为新的数组长度.
    • 该方法会改变原数组.
  5. reverse() 方法用于点到数组中元素的顺序
    • 该方法会改变原数组,不创建新的数组.
  6. shift() 方法删除并返回数组第一个元素
    • 返回值为第一个元素的值.
    • 若数组为空则返回undefined.
    • 该方法会改变原数组.
  7. slice() 方法从已有数组中返回选定元素
    • 接收2个参数start(必需)/end(可选).start规定从何处开始选取;end规定在何处结束.
    • 如果start/end为负数,则从数组末尾开始计算.
    • start与end制定的都是每个元素槽位的后侧边缘,即:

      //此处有图
    • 该方法不会改变原数组,返回一个新的数组.
  8. sort() 方法用于对数组元素排序
    • 接收参数sortby,规定排序的顺序.此参数必须是函数.若参数为空,则按照字符编码顺序进行排序.
    • 该方法会改变原数组,返回对原数组的引用,不生成副本.
    • 比较数有两个参数: a/b. a/b是数组中随机的元素.

      简而言之,

      return a - b : 升序排列

      return b - a : 降序排列
  9. splice() 方法向/从数组中添加/删除项目,然后返回被删除的项目.
    • 接收三种参数index(必需)/howmany(必需)/item1,item2,…(可选).index规定要操作的位置,howmany规定要删除的数量,item为要在该位置添加的新项目.
    • index为负数时,从数列末尾计算位置.如果howmany为0则不删除,即只添加item.
    • 返回值为被删除项目组成的数组.
    • 该方法会改变原数组.
  10. tostring() 方法把数组转换成字符串并返回结果.
    • 返回值与没有参数的join()方法返回的字符串相同.
    • 该方法会改变原数组.
  11. unshift() 方法可向数组开头添加一个或多个元素,并返回新的长度.
    • 接收1个或多个参数.
    • 返回值为新的数组长度
    • 该方法会改变原数组,不创建副本.

对于原数组不同操作的方法的横向比较

  1. 改变原数组的方法:
    pop() / push() / unshift() / reverse() / shift() / splice() / sort()
  2. 不改变原数组的方法:
    concat() / join() / slice()

CSS浮动与清除

浮动

  • 浮动的框可以向左或向右移动，直到它的外边缘碰到包含框或另一个浮动框的边框为止。
  • 由于浮动框不在文档的普通流中，所以文档的普通流中的块框表现得就像浮动框不存在一样。
  • 由于不在普通流中,浮动会导致父元素的高度坍塌
  • float属性:left/right/none/inherit

清除

  1. 添加额外标签

    缺点:添加无意义的标签,有违结构与表现的分离.
  2. 使用 br标签和其自身的 html属性

    缺点:添加无意义的标签,有违结构与表现的分离.
  3. 父元素设置 overflow：hidden

    缺点：内容增多时候容易造成不会自动换行导致内容被隐藏掉，无法显示需要溢出的元素.
  4. 父元素设置 overflow：auto 属性
  5. 父元素设置浮动
    缺点：使得与父元素相邻的元素的布局会受到影响，不可能一直浮动到body.
  6. 父元素设置display:table
    缺点：盒模型属性改变.

  7. 使用:after伪元素

    • U+200B 是一个Unicode字符里的“零宽度空格”,所以可以省略掉 visibility: hidden; ,替换为content:”200B”

BFC–Block Formatting Contexts （块级格式化上下文）

  1. 触发条件
    • float除none以外的值
    • overflow除visible以外的值(hidden/auto/scroll)
    • display = table/inline-block
    • position = absolute;fixed
  2. BFC的特性
    • BFC会阻止外边距增叠加
    • BFC不会重叠浮动元素
    • BFC通常可以包含浮动

px,em,rem & CSS选择器优先级

px/em/rem

  • px: 绝对大小.像素.
  • em: 相对长度单位,基于父级遗传字体大小计算.
  • rem: 相对长度单位,基于HTML根元素字体大小计算.
  • 任意浏览器的默认字体高都是16px.所有未经调整的浏览器都符合: 1em=16px.为了方便计算,可以将body中字体大小设置为font-size = 62.5%,则1em = 10px.rem同理.

Javascript的函数提示 & 回调函数 & 箭头函数

JavaScript的函数提升

  1. 函数语句随函数主体一起被提升到执行上下文的顶部.可以在函数创建之前使用作为语句创建的函数.

  2. 函数表达式在创建之前不能使用,只有声明部分会被提升,赋值发生在创建函数行.

Javascript的回调函数(callback())

函数A作为参数(函数引用)传递到另一个函数B中,且函数B会执行函数A,则A是B的回调函数.

Javascript的箭头函数

  1. 来自于ES6(ES2015)

  2. 形式:

    • 不引入参数

    • 引入1个参数

    • 引入多个参数

    • 如果函数体内有更多语句执行则不能省略return

    • 如果要返回对象,应(),以示和函数{}区分

  3. 箭头函数的this总是指向词法作用域

  4. 没有原型;没有参数对象;不能作为构造函数
