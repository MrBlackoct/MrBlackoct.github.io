解决项目中莫名其妙跳出多次弹窗提醒问题的过程

发表于 2018-01-09

解决项目中莫名其妙跳出多次弹窗提醒问题的过程

  1. 背景简介
    Project模块下有多个子模块,分别为newProject / penddingProject / myProject / projectInfo 等,其中整体模块由projectController.js控制,其余子模块有各自对应的Controller.各子模块由switchtab控制显示,不切换路由,各tab可同时存在于页面上.
    在newProject模块中,有表单保存功能,表单引用app-iframe组件.在加入保存成功后弹窗提醒功能后,测试中发现会弹出多次”保存成功”提醒,或两次,或四五次,甚至六七次.
    //使用AngularJS作为开发框架

  2. 逐步排查

    1. 首先检查alert的逻辑与数量.利用在newprojectcontroller.js中使用$scope.$on()监听接受广播消息SAVE_FORM_SUCCESS确认保存成功后弹窗提醒.检查后确认逻辑与数量都无误.

    2. 继续尝试弹出次数虽多但不定的情况.发现如果页面中存在多个子模块页面即存在多个app-iframe组件时提醒次数则会增多.初步判断可能是因为多个app-iframe之间会相互影响.检查app-iframe.js后发现广播SAVE_FORM_SUCCESS的机制只是判断了接收到的消息的类型,然后选择广播VALIDATE_FORM_SUCCESS / SAVE_FORM_SUCCESS.所以当页面中加载了多次app-iframe组件时就会广播多次该消息.

    3. 修改代码,为广播SAVE_FORM_SUCCESS部分加入判断.$rootScope.isListenedSavaSuccess为自定义变量,第一次判断时其一定为false,进入判断后将其置为true,则以后不会再进入改循环,然后正常进行广播事件的判断与执行.这样解决了调用一次app-iframe.js时会因为加载了多个app-iframe而出现多次广播的问题.

    4. 打断点继续测试,当加载多个app-iframe时确实只广播了一次SAVE_FORM_SUCCESS.但是发现仍然弹窗提醒两次,打断点发现alert部分被调用了两次.原因是newProject模块和myProject模块中的申请新流程页面都是加载newProjectController来实现的,所以$scope.$on()两次调用.

    5. 将$scope.$on()部分提升到projectController.js处,由于这是整个project模块的父级控制器,所以无论何时只会做出一次监听动作.至此问题解决.

  3. 总结
    遇到莫名其妙的问题后应根据其特征与规律入手,逐项检查,利用好断点调试可以监测程序的执行过程中你想要得到细节的部位的细节.

继承

约定父类:

1. 原型继承(将父类实例作为子类原型)

  • 实现

  • 特点 :

    1. 实例是父类和子类的共用实例
    2. 父类新增原型/实例方法,子类能都访问到
  • 缺点 :
    1. 要想为子类新增方法,必需要在new A()这样的语句前执行,不能放在构造器中
    2. 无法实现多继承
    3. 来自原型对象的引用属性是所有实例共享的
    4. 创建子类实例时,无法向父类构造函数传参

2. 构造继承(使用覅类构造函数来增强子类实例,等于赋值父类属性给子类)

  • 实现 :

  • 特点 :

    1. 解决了子类实例共享父类属性的问题
    2. 创建子类实例时可以向父类床底参数
    3. 可以实现多继承
  • 缺点 :
    1. 实例并不是父类的实例,只是子类的实例
    2. 只能击继承父类的实例属性和方法
    3. 无法实现函数复用,每个子类都有父类实例函数的副本,影响性能

3. 实例继承(为父类实例增加新特性,作为子类实例返回)

  • 实现 :

  • 特点 :

    1. 不限制调用方式,不管是new子类()还是子类(),返回的对象都具有相同效果
  • 缺点 :
    1. 实例是父类的实例,不是子类的实例
    2. 不支持多继承

4. 拷贝继承

  • 实现 :

  • 特点 :

    1. 支持多继承
  • 缺点 :
    1. 效率低,占用内存高
    2. 无法获取父类不可枚举的方法

5. 组合继承(通过父类构造,继承父类的属性并保留传参的特点.然后通过父类实例作为子类原型,实现函数复用)

  • 实现

  • 特点 :

    1. 弥补了方式2的缺陷,可以继承实例属性/方法,也可以继承原型属性/方法.
    2. 既是子类的实例,也是父类的实例
    3. 不存在引用属性共享问题
    4. 可传参
    5. 函数可复用
  • 缺点 :
    1. 调用了两次父类函数,生成了两份实例(子类实例覆盖了子类原型上的那份)

6. 寄生组合继承(通过寄生方式,砍掉父类属性)

  • 实现

  • 特点 :

    1. 堪称完美
  • 缺点 :
    1. 实现复杂

TIPS_1

tips1

函数原型链简单来说就是建立了变量查找机制.当访问一个对象的属性时,先查找对象本身是否存在,如果不存在,就去该对象所在的原型链上去找,直到Object对象为止.如果都没有找到该属性,则返回undefined.因此可以通过原型链来实现继承.

tips2

JS中的函数存在着三种角色:普通函数 / 构造函数 / 对象方法.同一个函数调用方式不同,函数的作用不同.
直接调用就是普通函数 / 通过new创建对象时就是构造函数 / 通过对象调用时就是方法.

tips3

原始类型存储在栈(stack)中.引用类型存储在堆(heap)中.存储在变量处的值是一个指针,指向存储对象的内存处(按址访问).

tips4

this指针存在于函数中,用以标识函数运行时所处的上下文.

HTML解析过程 & HTML离线缓存

HTML解析过程

  1. 用户输入网站,浏览器向服务器发出请求,服务器返回HTML文件.
  2. 渲染引擎开始解析HTML标签,并将标签转化为DOM节点,生成DOM树.
  3. 如果head中引用了外部CSS文件,发出CSS文件请求,服务器返回该文件,该过程会阻塞后面的解析.
  4. 如果引用了外部JS文件,则发出JS文件请求,服务器返回后立即执行该脚本,这个过程也会阻塞后面的解析.
  5. 引擎开始解析body里的内容,如果标签里引用了CSS样式,就需要解析刚才下载好的CSS文件,然后用CSS来设置标签的样式属性,并生成渲染树.
  6. 如果body中的img标签引用了图片资源,则立即向服务器发出请求,此时引擎不会等待图片下载完毕,而是继续解析后面的标签.
  7. 服务器返回图片文件,由于图片需要占用一定的空间,会影响到后面元素的排版,因此引擎需要重新渲染这部分内容.
  8. 如果此时JS脚本中运行了style.display = ‘none’, 布局被改变,引擎也需要重新渲染这部分代码,知道HTML结束标签为止,页面解析完毕.

HTML离线缓存

  • HTML5离线缓存又叫Application Cache,是从浏览器的缓存中分出来的一块缓存区.如果要在这个缓存中保存数据,可以使用一个描述文件(manifest file),列出要下载和缓存的资源.
  • manifest文件是简单的文本文件,它告之浏览器被缓存的内容(以及不缓存的内容).manifest文件可分为三个部分:
    • -CACHE MANIFEST-
      在此标题下列出的文件将在首次下载后进行缓存.
    • -NETWORK-
      在此标题下列出的文件需要与服务器连接,且不会被缓存.
    • -FALLBACK-
      在此标题下列出的文件规定当前页面无法访问时的回退页面(比如404页面).
  • 离线缓存为应用带来的优势:
    1. 离线浏览 — 用户可在应用离线时使用
    2. 速度 — 已缓存资源加载得更快
    3. 减少服务器负载 — 浏览器将只从服务器下载更新或更改过的资源

回流与重绘及其优化

发表于 2017-12-19

回流与重绘及其优化

  1. 回流(reflows)一定导致重绘(repaints)
  2. 何时发生reflows:
    • 添加或删除可见DOM元素
    • 元素位置改变
    • 元素尺寸改变 — 边距/边框/填充/宽/高
    • 内容改变 — 比如改变文本或图片大小而引起的计算值宽高改变
    • 页面渲染初始化
    • 浏览器窗口尺寸改变 — resize事件发生时
  3. 如何减少reflows & repaints
    减少reflows/repaints其实就是需要减少对render tree的操作(合并多次DOM和样式修改),并减少对一些style信息的请求.
    • 直接改变className,如果动态改变样式,则使用CSSText.
    • 让要操作的元素进行”离线处理”,处理完后一起更新.
      1. 使用DocumentFragment进行缓存操作,引发一次回流和重绘.
      2. 使用display : none,一发两次回流和重绘.
      3. 使用cloneNode(true/false)和replaceChild引发一次回流和重绘.
    • 不要经常访问会引起浏览器flush队列的属性.如果确实要访问,利用缓存.
    • 让元素脱离动画流,减少回流的render tree的规模.

CSS的display & box-sizing & position & 块级元素和行内元素

发表于 2017-12-19

CSS的display & 块级元素和行内元素

display =

  1. display = none
    元素被隐藏.此时元素脱离文档普通流,不占据空间.
    • 与visibility = hidden的区别
      1. visibility =hidden只是让元素不可见,仍占据位置,仍在文档普通流中.
      2. 改变display会导致reflows从而也导致repaints.改变visibility只会导致repaints
      3. display为非继承属性;visibility为可继承属性.
  2. display = block
    元素以块级显示.块级元素的前后都有换行符
    • 块级元素会自动继承父元素的不影响DOM树只影响渲染树的属性.
    • 子元素如果没有设置width属性,那么与父元素同宽,但这不是继承,是块级元素本身要占据最大空间的性质.
  3. display = inline
    将元素设置为行内元素,可以在同一行显示.
    • 行内元素无法设置height/width属性,根据内容选择.
      • 解决方法:
        使用line-height,计算line-height与font-size的差值,分为两半加到一个文本行内容的顶部和底部.
    • 行内元素不可设置外上下边距,可以设置外左右边距;可以设置所有内边距.
  4. display = inline-block
    将元素显示在行内,但可以设置其上下外边距,可以设置height/width.
  5. display = inherit
    继承父元素display属性.

box-sizing =

  1. content-box : 宽高应用到元素的内容框,在宽高之外绘制内边距/边框.
  2. border-box : 宽高包括了内边距和边框,在其内部绘制内容.
  3. inherit : 继承.

position =

  1. absolute : 生成据对定位的元素,相对于第一个static以外的父元素进行定位,通过left/right/top/bottom属性进行规定.
  2. fixed : 生成绝对定位的元素,相对于浏览器窗口进行定位,通过left/right/top/bottom属性进行规定.
  3. relative : 生成相对定位的元素,相对于其他正常位置进行定位.
  4. static : 默认值.没有定位,元素出现在正常的流中.(忽略top/bottm/left/right).
  5. inherit : 继承.

块级元素/行内元素 — block element / inline element

  • 块级元素的特点 :
    • 每个块级元素独占一行(float除外).
    • 可以设置height/width/margin/padding.
    • 宽度默认占满容器.
    • 可以容纳其它块级元素和内联元素
  • 内联元素的特点 :
    • 和其它元素在同一行.
    • 不能设置height/width,margin和padding只能设置左右.
    • 只能容纳其它内联元素.
  • 常见块级元素 :
    div / dl / p / ul / h1-6 / form/ ol / menu / table
  • 常见内联元素
    a / br / input / img / lable / span / select

悲观锁/乐观锁 & XSS & 云服务 & webpack的作用

发表于 2017-12-19

悲观锁/乐观锁 & XSS & 云服务 & webpack的作用

悲观锁/乐观锁

  • 悲观锁 :
    • 每次读取文件时加锁,不允许别人操作.
    • 缺点 : 如果操作时间太长,等待会很久.
  • 乐观锁 :
    • 使用版本号,若操作结束后与操作前版本号不同则放弃更改重新操作;若版本号相同则保留更改并将版本号加一.
    • 缺点 : 如果使用在冲突过多处反而会增加消耗.

XSS(跨站脚本攻击)

  1. 什么是XSS
    恶意攻击者往Web页面里插入恶意Script代码,当用户浏览该页之时,嵌入其中Web里面的Script代码会被执行,从而达到恶意攻击用户的目的.
  2. XSS的类型
    • 存储型 : 持久化,代码存储在服务器中.容易造成蠕虫/盗窃cookie等.
    • 反射型 : 非持久化,需要欺骗用户点击链接才能触发代码.

三种云服务

  • SaaS — Software-as-a-Service
    软件即服务 – 处于顶层
  • PaaS — Platform-as-a-Service
    平台即服务 – 处于中间层
  • IaaS — Infrastructure-as-a-Service
    基础设施即服务 – 处于底层

webpack的作用

webpack主要的功能就是模块化 & 处理依赖 & 打包.

  • 一切皆模块,达到重复利用等目的.
  • 异步加载部分代码以实现按需加载.
